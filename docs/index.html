Features

Everything you need,
nothing you don't

TAK is designed to do one thing exceptionally well: turn your voice into text, instantly, privately, and everywhere.

100% Offline

All transcription happens locally on your machine. No cloud APIs, no data leaving your computer, no internet required.

GPU Accelerated

Blazing fast inference with CUDA on Linux (NVIDIA) and Metal on macOS (Apple Silicon). CPU fallback available.

System-Wide

Works in any application with a text cursor — browsers, editors, terminals, Slack, email, everywhere.

Push-to-Talk

Microphone only active while you hold the trigger key. No always-on listening, no accidental recordings.

Bilingual

Auto-detects English and Spanish. Seamlessly switch languages mid-conversation without configuration.

Configurable

Choose your trigger key, model size, input method, and audio device. Tune the balance of speed vs. accuracy.

Native Menu Bar

Lives in your macOS menu bar with recording status, quick access to Preferences, and a clean uninstall option.

Preferences UI

Graphical settings panel for trigger key, model, audio device, and clipboard mode. Models download in-app with a progress bar.

Visual Feedback

A floating recording pill appears on screen while you speak, turning yellow during transcription. Always know what TAK is doing.

How It Works

Three stages, one keystroke

TAK runs a tight loop: listen, transcribe, type. The entire pipeline completes in under a second.

01

Record

Hold the trigger key. TAK captures audio from your microphone via Core Audio (macOS) or PipeWire/ALSA (Linux). Audio is resampled to 16 kHz mono and auto-normalized.

pynput sounddevice Core Audio PipeWire
02

Transcribe

On key release, Whisper runs inference on your GPU. Voice activity detection filters silence. Language detection handles bilingual input automatically.

OpenAI Whisper faster-whisper mlx-whisper Silero VAD
AI
03

Type

Transcribed text is injected into the focused window via simulated keystrokes (Linux) or clipboard paste (macOS). It just appears where your cursor is.

xdotool AppleScript Clipboard

Architecture

Modular by design

TAK uses dependency injection with platform-specific backends. The core is platform-agnostic — backends are pluggable.

Entry Point
__main__.py Platform detection & dependency injection
Shared Core
app.py TakApp, base classes, CLI, audio utilities
TakApp BaseAudioRecorder BaseTranscriber parse_args() _resample() KEY_MAP
Platform Backends
Linux
platforms/linux.py
LinuxAudioRecorder LinuxTranscriber faster-whisper + CUDA xdotool / xclip
macOS
platforms/macos.py
MacAudioRecorder MacTranscriber mlx-whisper + Metal AppleScript / pbcopy
Zero platform branching in core

All if IS_MACOS logic lives in __main__.py only

Constructor injection

TakApp receives backends as arguments — never imports platform modules

Self-contained backends

Deleting linux.py on macOS causes zero errors, and vice versa

Lazy imports for heavy deps

faster_whisper and mlx_whisper imported inside __init__()

From Source

Install from source (macOS & Linux)

Terminal 
$ git clone https://github.com/lchonkan/tak.git
$ cd tak
$ ./install.sh

The installer auto-detects your platform, installs dependencies, creates the conda environment, and sets permissions.

Run 
$ ./run.sh
Terminal
# Install system dependencies
$ brew install portaudio

# Clone and set up
$ git clone https://github.com/lchonkan/tak.git && cd tak
$ conda create -n tak python=3.11 -y
$ conda activate tak
$ pip install -r requirements-macos.txt

# Grant Accessibility permission
# System Settings → Privacy & Security → Accessibility → Terminal

# Run
$ ./run.sh
Terminal
# Install system dependencies
$ sudo apt install xdotool xclip libportaudio2

# Clone and set up
$ git clone https://github.com/lchonkan/tak.git && cd tak
$ conda create -n tak python=3.11 -y
$ conda activate tak
$ pip install -r requirements-linux.txt

# Grant input device access
$ sudo usermod -aG input $USER
# Log out and back in

# Run
$ ./run.sh

Configuration

./run.sh --key caps_lock Use Caps Lock as trigger
./run.sh --model large-v3 Maximum accuracy
./run.sh --model tiny Maximum speed
./run.sh --clipboard Paste via clipboard
./run.sh --cpu CPU-only mode
./run.sh --device 2 Specific audio device
Under the Hood

Threading model

TAK uses three threads to keep the key listener responsive while transcription runs in the background.

Main Thread
Init app.run() — keyboard listener loop
Listener Thread
on_press Recording on_release
Worker Thread
Transcribe Type

A threading lock ensures only one transcription runs at a time. New key presses during transcription are queued.
